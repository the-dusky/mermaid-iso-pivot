# arch-explorer Documentation

## Structure

### [External Documentation](./external/)
User-facing documentation following the [Diataxis](https://diataxis.fr/) framework:
- **Tutorials** - Learning-oriented lessons
- **How-to Guides** - Task-oriented recipes
- **Reference** - Technical specifications
- **Explanation** - Conceptual discussions

### [Internal Documentation](./internal/)
Contributor and development documentation:
- **ADRs** - Architecture Decision Records (feature plans)
- **Dev Notes** - Implementation notes and research

## Development Process

We follow a **plan-first** approach:

1. **Write an ADR** before implementing any significant feature
2. **Review the ADR** for issues and edge cases
3. **Fix the plan** if problems are found
4. **Implement** once the plan is solid

ADRs are living documents that evolve as we learn. Update them when reality differs from the plan.
